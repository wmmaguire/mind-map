feat(auth): password reset email flow, reset page, and dev server hardening

Password reset (1h one-time token, SHA-256 stored on User)
- POST /api/auth/forgot-password, POST /api/auth/reset-password; reset clears fields and sets mindmap_auth cookie
- User: passwordResetTokenHash, passwordResetExpiresAt; authService token helpers + tests
- Outbound mail: server/lib/passwordResetMail.js (nodemailer); SMTP_URL must be smtp:// or smtps:// (https app URLs ignored)
- Optional env: APP_PUBLIC_ORIGIN, SMTP_*, MAIL_FROM; production requires SMTP + APP_PUBLIC_ORIGIN for forgot-password to persist tokens
- Client: GuestIdentityBanner forgot-password UI; PasswordResetPage at /reset-password; AuthContext requestPasswordReset + completePasswordReset

Server / dev UX
- http.createServer with maxHeaderSize 64KB (HTTP 431); optional HTTP_MAX_HEADER_SIZE
- Root npm run dev: NODE_OPTIONS=--max-http-header-size=65536
- GET /health and GET /api/test registered early; startup log lists /api/test path correctly
- client npm start: PORT=3000 so CRA does not bind the same port as the API when shell exports PORT=5001
- Client config: development API base http://localhost:5001 (CORS); apiRequest network error text updated
